Data Channel Protocol

Once connected, a single WebRTC data channel named "data" carries everything. Every message is JSON published directly to the robot's internal ROS 2 topics.

format
{ "type": <channel-type>, "topic": <ros2-topic>, "data": <payload> }
type
"req" command request · "msg" plain publish · "vid" / "aud" media toggles · "validation" handshake · "heartbeat"
topic
A ROS 2 topic on the robot, e.g. rt/api/sport/request for motion, rt/utlidar/switch for LiDAR, rt/uslam/client_command for SLAM
data
Request envelope with header.identity.api_id selecting the action and parameter carrying arguments. The robot acts on it immediately.
Validation
Robot sends a challenge string. Client replies with base64(md5("UnitreeGo2_" + challenge)). Then publish "on" to start media streams.
Commands
Sport (move, tricks, flips, gaits), SLAM (map, localize, navigate, patrol), audio, LiDAR, obstacle avoidance. All through api_id enums on topics.
Key Insight
The dog is a ROS 2 robot with a WebRTC bridge. Hold the data channel, and you have the same control surface the phone app has.

Autonomy Architecture: Project ROSE

Six-layer architecture for an autonomous gallery docent. Each layer is an independent module built and tested separately. The orchestrator ties them together.

L6
Orchestrator (the "mind")
Explicit state machine. Owns all transitions. L2 through L5 are subordinate services it coordinates.
PATROL → GREET → CONVERSE → RETURN_BASE → IDLE
L5
Self-Care Supervisor
Watchdog on battery SOC, motor temps, BMS/IMU temps. Forces return-to-base and signals a human. Highest priority.
L4
Conversation (Voice Agent)
Gemini Live API, headless voice. Dog's mic is input, speaker is output. Per-gallery knowledge injected by current waypoint.
L3
Perception and Triggers
MediaPipe Gesture Recognizer (trained model, not geometry rules) with confidence floor + temporal stability gate. Visitor-present detector triggers conversation.
L2
Patrol / Navigation Manager
Holds ordered waypoint list. Issues navigation goals to the robot's onboard SLAM. Exposes pause() and resume() for interrupts from L3/L5/L6.
L1
Map and Waypoints (one-time setup)
Walk the space once with Unitree's built-in SLAM. Define named waypoints (galleries, base). Output is a config artifact, not runtime code.
Priority
Self-Care > Visitor Interaction > Patrol
Control Bus (existing infrastructure)
signaling_bridge.py + WebRTC data channel. All layers talk to the robot through this. Already built and working.
